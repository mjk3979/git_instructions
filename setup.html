Setting up a repository

Setting your name and email

Before using Git on any system for the first time, it is a good idea to set your name and email address, since Git will include this information in each commit message you author. Follow these steps:

  1. Set your name by using this command:
    $ git config --global user.name "<Your Name>"
  2. Set your email as follows:
    $ git config --global user.email <your_email_address>

If you plan to develop on the CS systems

If you intend to write your code on the department's systems (e.g. while connected via SSH), all you need to do is initialize a standard Git repository. To do this, follow these steps.

  1. Make sure you've set your name and email, as detailed in the above section. To check, look for this information in the output of the command:
    $ git config --global -l
  2. Move into the folder where you keep your work for the course, for example by running the command:
    $ cd ~/Courses/CS/CS1/Labs
  3. Create a repository in a new folder of your choice. For instance, to create one called Lab1, run:
    $ git init Lab1
  4. To check that your repository was successfully created, use the following command to list the contents of the current directory:
    $ ls -a
    and look for the hidden .git folder. This folder identifies the directory as a Git repository.

If you plan to develop on your own machine

If you intend to write your code on your own system, create an upstream on the CS systems and clone it to a standard Git repository on your own machine. This is done as follows:

  1. Working on a CS department system, move into the folder where you keep your work for the course, for example by running the command:
    $ cd ~/Courses/CS/CS1/Labs
  2. Create a bare repository in a new folder of your choice. For instance, to create one called Lab1.git, run:
    $ git init --bare Lab1.git
  3. On your own system, download Git from the project website at http://git-scm.com/downloads and install the package
  4. If you're on a Windows system, locate and launch the newly-installed application called Git Bash; otherwise, simply open a terminal (e.g. on Mac OS X, click the magnifying glass in the top right corner of the screen and type "Terminal").
  5. In the terminal on your own system, move into the directory where you keep your coursework, for example:
    $ cd ~/Courses/CS/CS1/Labs
  6. Clone the repository you created earlier on the CS systems. To do this for an upstream you called Lab1.git, run:
    $ git clone @glados.cs.rit.edu:Courses/CS/CS1/Labs/Lab1
  7. To check that your repository was successfully created, use the following command to list the contents of the current directory:
    $ ls -a
    and look for the hidden .git folder. This folder identifies the directory as a Git repository.

Regarding code hosting services and public repositories

There are many online services that offer Git repository hosting, the most popular being GitHub and BitBucket. The CS department recommends that you store your code on department systems. If, however, you decide to use a third-party service, carefully ensure that your hosted repositories are private. If you publish course code to a public repository, you can be penalized for cheating pursuant to RIT's Academic Integrity Policy and the department Policy on Academic Honesty.
