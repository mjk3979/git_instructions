Introduction to Git

What is version control?

Version control is used to keep track of the changes made on a set of files over time (commonly known as "revisions" or "commits"). This is useful for recovering from accidentally deleted or broken code and for collaborative coding in a team. Git is a good example of a version control system.

Why should I use it?

(Because we said so.)

How does Git fit in?

Each Git repository consists of a .git directory containing a representation of the version history. This directory is usually located inside a working directory that contains the currently "checked-out" commit, as well as any unversioned changes that have been made.

Each "change" (or commit) consists of a user-entered message describing the changes that were made, a representation of the changes themselves, and a pointer to the previous version of each file. In this way, Git creates something like a linked-list of commits that shows the evolution of a project over time.
